# BUG-006: ReleaseHealthCheckPage — "Onayla" butonu 404 hatası veriyor

## Sorun

Release Health Check sayfasındaki "Onayla / Approve" butonu 404 döndürüyor. Sürümü onaylamak mümkün değil.

## Adımlar (Reproduce)

1. Login yap (admin veya release manager rolü gerekli)
2. Sol menüden "Release Health Check" sayfasına git
3. Bir ürün ve versiyon seç
4. "Onayla" butonuna tıkla

## Beklenen Davranış

Versiyonun phase'i güncellenmeli, onay kaydedilmeli.

## Gerçekleşen Davranış

404 hatası.

## Teknik Kök Neden

**Satır 107:**
```typescript
mutationFn: () => apiClient.post(`/product-versions/${selectedVersion}/approve`),
```

Backend'de `/approve` endpoint'i yok. Mevcut endpoint:
```
PATCH /api/product-versions/:id/phase
```

## Önerilen Fix

```typescript
// L107 — POST /approve → PATCH /phase
mutationFn: () => apiClient.patch(`/product-versions/${selectedVersion}/phase`, { phase: 'APPROVED' }),
```

Not: Backend'de hangi phase değeri "onay" anlamına geliyor doğrulanmalı. `productVersions.routes.ts`'de geçerli phase değerleri kontrol edilmeli.
