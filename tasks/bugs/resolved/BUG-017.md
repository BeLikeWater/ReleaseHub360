# BUG-017: Hedef versiyon seçicisi düşük versiyonları filtrelemiyor — İş Kuralı 2 ihlali

## Sorun

İş Kuralı 2: "Hedef Versiyon: Müşterinin almak istediği versiyon. Yalnızca kaynak versiyondan yüksek versiyonlar seçilebilir." Mevcut implementasyonda hedef versiyon dropdown'u yalnızca kaynak versiyon ile **aynı** olan versiyonu filtreler, kaynak versiyondan düşük olanları dahil etmeye devam eder. Kullanıcı kaynak olarak `v2.1.0`, hedef olarak `v1.0.0` seçebilir; bu mantıksal olarak anlamsız bir sync işlemi başlatır.

## Adımlar (Reproduce)

1. Ürün seç → birden fazla versiyonu olan bir ürün (ör. v1.0.0, v1.5.0, v2.1.0)
2. Kaynak Versiyon: `v2.1.0` seç
3. Hedef Versiyon dropdown'una bak
4. `v1.0.0` ve `v1.5.0` hâlâ seçilebilir durumda

## Beklenen Davranış

Hedef versiyon dropdown'u yalnızca kaynak versiyondan **semantik olarak büyük** versiyonları listelemelidir. `v2.1.0` kaynak seçiliyse dropdown boş olabilir (veya hiç seçenek olmaz diye disabled).

## Gerçekleşen Davranış

```tsx
// Mevcut kod (CodeSyncPage.tsx ~L254):
{versions.filter(v => v.id !== sourceVersionId).map(v => ...)}
```
Yalnızca kaynak versiyonun kendisi çıkarılıyor; diğer tüm versiyonlar (düşük olanlar dahil) gösteriliyor.

## Teknik Kök Neden

Versiyon string'leri semantik versiyonlama formatında (`v1.0.0`, `v2.1.0` vb.) veya farklı formatlarda olabilir. `semver` paketi kullanılarak ya da basit string karşılaştırma ile (uyarı: `v10.0.0 > v9.0.0` string sort'ta yanlış sonuç verebilir) filtreleme yapılabilir.

**Çözüm önerisi:**
```tsx
import semver from 'semver';

// Hedef versiyon filter:
versions
  .filter(v => v.id !== sourceVersionId)
  .filter(v => {
    const srcV = versions.find(x => x.id === sourceVersionId);
    if (!srcV) return true;
    const clean = (s: string) => s.replace(/^v/, '');
    return semver.valid(clean(v.version)) && semver.valid(clean(srcV.version))
      ? semver.gt(clean(v.version), clean(srcV.version))
      : true; // parse edilemezse göster (güvenli fallback)
  })
  .map(v => <MenuItem .../>)
```
