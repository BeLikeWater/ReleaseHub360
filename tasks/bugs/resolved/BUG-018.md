# BUG-018: Geçmiş sekmesinde "alınan workitem sayısı" gösterilmiyor — AC-26 eksik

## Sorun

AC-26: "Geçmiş sekmesi: tarih, versiyon aralığı, **alınan workitem sayısı**, durum ve PR linki içerir." Mevcut implementasyonda geçmiş kartı yalnızca PR sayısını gösteriyor (`{r.payload?.prIds?.length ?? '?'} PR`). Workitem sayısı hiç gösterilmiyor.

## Adımlar (Reproduce)

1. En az 1 sync tamamla (SUCCESS)
2. "Geçmiş" sekmesine geç
3. Kart içeriğini incele

## Beklenen Davranış

Geçmiş kartı şunu göstermeli:
```
v2.0.0 → v2.1.0 · 3 workitem, 12 PR
```

## Gerçekleşen Davranış

Şu an şunu gösteriyor:
```
[UUID] → [UUID] · 12 PR
```
Workitem sayısı kart üzerinde hiç görünmüyor.

## Teknik Kök Neden

`SyncHistory.payload` JSON nesnesi `prIds[]` field'ını tutuyor. Workitem sayısı ise `workitemSummary` string olarak payload'a geçirilmiş olabilir ancak count olarak saklanmıyor. Ayrıca BUG-016 ile aynı UUID sorunu ektedir.

**Çözüm önerisi (Backend):** `POST /start` route'u payload'a workitem count da eklesin:

`codeSync.routes.ts`:
```typescript
// body'den workitemIds veya frontend'den selWiCount gönderilmeli
payload = { ...payload, workitemCount: body.workitemCount ?? null };
```

Frontend'de `startSyncMutation`:
```typescript
workitemCount: selWiCount,
```

`HistoryTab`'da:
```tsx
{r.payload?.workitemCount ?? '?'} workitem, {r.payload?.prIds?.length ?? '?'} PR
```

Alternatif: `workitemSummary` string'inden `•` karakteri sayısıyla count çıkarılabilir (hacky, önerilmez).
