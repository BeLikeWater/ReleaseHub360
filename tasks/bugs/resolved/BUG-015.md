# BUG-015: Sync onay dialogunda oluşturulacak feature branch adı gösterilmiyor

## Sorun

AC-18, "Sync Başlat" onay dialogunda **oluşturulacak branch adı** bilgisinin gösterilmesini zorunlu kılıyor. Mevcut implementasyonda dialog, müşterinin mevcut branch'ini (`selBranch?.branchName`, ör. `pre_master`) gösteriyor. Ancak sistem yeni bir `sync/v{hedef-versiyon}-{servis-adı}-{tarih}` branch'i açacak — bu branch adı dialogda hiç gösterilmiyor. Kullanıcı hangi branch'in oluşturulacağını bilmiyor.

## Adımlar (Reproduce)

1. Ürün → Servis → Müşteri Branch seç
2. Kaynak ve hedef versiyonu seç → "Delta Yükle"
3. En az 1 PR seç → "Sync Başlat" butonuna tıkla
4. Açılan onay dialogunu incele

## Beklenen Davranış

Dialog şu bilgileri içermeli:
- **Hedef branch (müşteri):** `pre_master`
- **Oluşturulacak sync branch:** `sync/v2.1.0-ETX.API.Gateway-20260225`
- Versiyon aralığı ve PR/workitem sayısı

## Gerçekleşen Davranış

Dialogda yalnızca müşteri branch adı (`pre_master`) ve repo adı gösteriliyor. Sistem tarafından oluşturulacak `sync/v...` branch'ının adı, format veya benzeri hiçbir bilgi yok.

## Teknik Kök Neden

`CodeSyncPage.tsx` (~L447–L475), confirm dialogunda `selBranch?.branchName` ve `selBranch?.repoName` değişkenlerini kullanıyor. Sync branch adı backend'de `buildSyncBranchPrefix()` ile üretiliyor ve MCP tarafından finalize ediliyor; frontend sync başlamadan önce bu ismi bilmiyor.

**Çözüm önerisi:** `buildSyncBranchPrefix` mantığını frontend'de de üret ve dialogda preview olarak göster:
```tsx
const expectedSyncBranch = `sync/v${tgtVer?.version}-${selectedService?.repoName ?? selectedService?.name}-${new Date().toISOString().slice(0, 10).replace(/-/g, '')}`.toLowerCase();
```
Bu değeri dialog body'sine ekle:
```
Oluşturulacak Branch: sync/v2.1.0-etx.api.gateway-20260225
Hedef Branch: pre_master
```
