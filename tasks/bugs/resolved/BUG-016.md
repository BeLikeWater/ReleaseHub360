# BUG-016: Geçmiş sekmesinde versiyon aralığı UUID gösteriyor, insan okunabilir versiyon adı yok

## Sorun

AC-26, geçmiş sekmesinin "versiyon aralığı" bilgisini içermesini zorunlu kılıyor. Mevcut implementasyonda `r.payload?.sourceVersionId` ve `r.payload?.targetVersionId` field'ları kullanılıyor; ancak bu field'lar `SyncHistory.payload` JSON objesinde **versiyon UUID**'lerini tutuyor (ör. `cm8xx2a3v00001...`), versiyonun okunabilir adını değil (ör. `v2.1.0`).

## Adımlar (Reproduce)

1. Bir sync tamamla (SUCCESS, FAILED veya CONFLICT)
2. "Geçmiş" sekmesine geç
3. Geçmiş kartında versiyon aralığı alanını incele

## Beklenen Davranış

Geçmiş kartı şunu göstermeli:
```
v2.0.0 → v2.1.0 · 12 PR
```

## Gerçekleşen Davranış

Şu an şunu gösteriyor:
```
cm8xx2a3v0000lq1kh3ya → cm8yy9xz000023kq5mn · 12 PR
```
(`sourceBranch` ve `targetBranch` field'larına fallback yapıldığında ise `master → master` gibi branch adları gösterilebilir — bu da yanlış.)

## Teknik Kök Neden

`HistoryTab` bileşeni (`CodeSyncPage.tsx` ~L650):
```tsx
{r.payload?.sourceVersionId ?? r.sourceBranch} → {r.payload?.targetVersionId ?? r.targetBranch}
```
`payload.sourceVersionId` ve `payload.targetVersionId`, backend `POST /start` route'undan yazılıyor ancak bunlar ProductVersion'ın `id` (UUID) alanları.

**Çözüm önerisi (2 seçenek):**

**A — Backend fix (önerilen):** `POST /start` route'u payload'a `sourceVersionStr` ve `targetVersionStr` alanlarını da eklesin:
```typescript
// codeSync.routes.ts /start route
const [srcVer, tgtVer] = await Promise.all([
  prisma.productVersion.findUnique({ where: { id: sourceVersionId }, select: { version: true } }),
  prisma.productVersion.findUnique({ where: { id: targetVersionId }, select: { version: true } }),
]);
payload = { ...payload, sourceVersionStr: srcVer?.version, targetVersionStr: tgtVer?.version };
```

**B — Frontend fix:** `HistoryTab` component'ine `versions` prop geç ve UUID'leri lookup et.
