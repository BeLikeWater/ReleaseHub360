## Semptom

Appwys.io + v1.2.1 seçilince Release Health Check sayfasında BOM ve PR bölümleri boş görünüyor.
Dün aynı seçimle release'ler arası PR'lar listelendi. Herhangi bir hata mesajı gösterilmiyor.

## Kök Sebep

`GET /api/tfs/release-delta` endpoint'i her servis için `vsrm.dev.azure.com` üzerinden
Azure Classic Release tanımlarını çekiyor. Ancak:

```
HTTP 403 — VS403463: The conditional access policy defined by your
Microsoft Entra administrator has failed.
```

**Entra Conditional Access Policy (CAP)** backend'in çalıştığı ağdan `vsrm.dev.azure.com`'a
erişimi engelliyor. PAT geçerli ve scope'u var; problem ağ/cihaz politikası.

### Neden Dün Çalıştı?

- Backend muhtemelen VPN'deyken ya da CAP'nin aktif olmadığı bir ağdan çalışıyordu.
- Entra admin'in CAP'ı yakın zamanda değiştirmiş olması mümkün.

## İkincil Kod Hataları (Kod Düzeyinde)

1. **Silent fail:** `if (!r.ok) return;` ile 403 durumunda servis sessizce atlanıyor,
   kullanıcıya herhangi bir hata/uyarı gösterilmiyor.

2. **Timeout yok:** `fetch()` çağrıları `AbortController` içermiyor.
   Azure yavaş yanıt verirse endpoint sonsuza kadar askıda kalıyor.

## Etkilenen Akış

```
GET /api/tfs/release-delta?productId=c6d27be4-...
  → 6 servis: /release/definitions?searchText=... → vsrm.dev.azure.com → 403 CAP
  → if (!r.ok) return;  (sessiz skip)
  → results [] (boş)
  → { data: [] } 200 OK dönüyor
  → BomSection deltaMap empty → UI'da PR yok, hata yok
```

## Gerekli Düzeltmeler

### Fix 1 — Backend: Fetch Timeout (tfs.routes.ts)

`release-delta` handler içindeki tüm `fetch()` çağrılarına 15 saniyelik
`AbortController` timeout ekle. Infinite hang önlenir.

### Fix 2 — Backend: 403 / CAP Hatasını Surface Et (tfs.routes.ts)

`if (!r.ok) return;` yerine 401/403'te `authError` flag'i set et,
response'a ekle:

```json
{ "data": [], "authError": "vsrm 403 — Entra CAP engelledi. Backend VPN'de mi?" }
```

### Fix 3 — Frontend: Hata Alert Göster (ReleaseHealthCheckPage.tsx)

`BomSection` içinde `deltaRaw.authError` varsa MUI `Alert` ile göster:
```
⚠️ Azure Release (VSRM) erişimi engellendi.
Backend sunucusu şirket VPN'ine bağlı mı?
```

### Fix 4 — Altyapı / Kullanıcı Aksiyonu

Backend sunucusu Microsoft Entra CAP koşullarını sağlayan ağdan çalışmalı:
- Şirket VPN'i aktif et
- Veya Azure'da barındırılan bir sunucu kullan (compliant network)
- Veya Entra admin'e CAP muafiyeti için başvur
