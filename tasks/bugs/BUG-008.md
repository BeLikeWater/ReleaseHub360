# BUG-008: ReleaseHealthCheckPage — Blocker Todo hesabı yanlış alan kullanıyor (status vs isCompleted)

## Sorun

Release Health Check sayfasında "Blocker Todo (P0)" sayacı ve sağlık skoru hesabı, TodoDone durumunu
yanlış alanla kontrol ediyor. Bu nedenle tamamlanmış P0 todo'lar bile hâlâ "blocker" olarak sayılıyor
ve sağlık skoru hatalı düşük gösterilebiliyor.

## Adımlar (Reproduce)

1. Bir versiyona en az 1 tane P0 priority'li ReleaseTodo ekle (API veya seed)
2. O todo'yu tamamla (`isCompleted: true` olarak PATCH et)
3. Release Health Check sayfasına git → Aynı versiyonu seç
4. "Blocker Todo (P0)" sayacının 0 göstermesi gerekirken hâlâ 1 gösterdiğini gözlemle
5. Sağlık skoru haksız yere -5 düşük hesaplandığını gözlemle

## Beklenen Davranış

`isCompleted: true` olan P0 todo'lar blocker sayılmamalı, sağlık skoru buna göre hesaplanmalı.

## Gerçekleşen Davranış

Tüm P0 todo'lar, tamamlanmış olsa da blocker olarak sayılıyor.

## Teknik Kök Neden

`ReleaseHealthCheckPage.tsx` içindeki `Todo` tipi:
```typescript
type Todo = { id: string; title: string; priority: string; status: string };
```

Ve blocker kontrolü:
```typescript
const blockerTodos = todos.filter(t => t.priority === 'P0' && t.status !== 'DONE');
```

Oysa backend (`prisma/schema.prisma` satır 281) ve `/api/release-todos` endpoint'i şu alanları döndürüyor:
```json
{ "id": "...", "priority": "P0", "isCompleted": true }
```

`status` alanı **yok**, sadece `isCompleted: boolean` var.
`t.status` her zaman `undefined` döneceğinden `undefined !== 'DONE'` = `true` olur.
Yani **bütün P0 todo'lar her zaman blocker sayılır**.

## Önerilen Fix

`packages/frontend/src/pages/ReleaseHealthCheckPage.tsx`:

```typescript
// Tip düzeltmesi (satır ~18):
// ÖNCE:
type Todo = { id: string; title: string; priority: string; status: string };
// SONRA:
type Todo = { id: string; title: string; priority: string; isCompleted: boolean };

// Blocker hesabı (satır ~116):
// ÖNCE:
const blockerTodos = todos.filter(t => t.priority === 'P0' && t.status !== 'DONE');
// SONRA:
const blockerTodos = todos.filter(t => t.priority === 'P0' && !t.isCompleted);
```

## Etki

- Sağlık skoru hatalı hesaplanıyor (gerçekte hazır release'ler "hazır değil" görünebilir)
- Todo tamamlansa da Blocker sayısı azalmıyor
- Veri yokken sessiz bug — todo eklendikçe manifest oluyor

---
> **Frontend Developer**: `Todo` tipi + 1 satır filter değişikliği. `ReleaseTodosPage.tsx` doğru (`isCompleted`) kullanıyor, bu sayfada da aynı alan kullanılmalı.
