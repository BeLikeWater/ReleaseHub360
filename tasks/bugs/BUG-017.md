# BUG-017: CustomerDashboardPage — Artifact download düğmeleri her zaman devre dışı

## Sorun

`ProductCard` bileşenindeki **HelmChart İndir** ve **Paketi İndir** düğmeleri, backend'de
yeni oluşturulan `/api/customer-deployments/download/helm/:mappingId` ve
`/api/customer-deployments/download/binary/:mappingId` endpoint'lerini **kullanmıyor**.

Bunun yerine, `versionPackages` dizisindeki `VersionPackage.artifactUrl` alanına bakıyorlar.
Bu alan büyük ihtimalle DB'de `null` — yeni seed'lenmemiş ürünler için doldurulan bir alan değil.

Sonuç: Düğmeler `disabled={!helmPackage}` / `disabled={!binaryPackage}` korumasıyla
**her zaman devre dışı** kalıyor. Kullanıcı hiçbir zaman dosya indiremez.

## Etkilenen Kod

```tsx
// packages/frontend/src/pages/CustomerDashboardPage.tsx ~line 356-375
const helmPackage = mapping.versionPackages.find(p => p.packageType === 'HELM');
const binaryPackage = mapping.versionPackages.find(p => p.packageType === 'BINARY' || p.packageType === 'INSTALLER');

// HELM button (line ~374):
onClick={() => handleDownload(helmPackage?.artifactUrl ?? null)}
disabled={!helmPackage}

// BINARY button (line ~387):
onClick={() => handleDownload(binaryPackage?.artifactUrl ?? null)}
disabled={!binaryPackage}
```

## Adımlar (Reproduce)

1. Müşteri dashboard sayfasını aç (`/customers/:id`)
2. `deploymentModel: 'ON_PREM'`, `artifactType: 'HELM'` olan bir ürün kartına bak
3. **HelmChart İndir** düğmesinin `disabled` olduğunu gözlemle

## Beklenen Davranış

- Düğmeye tıklandığında `/api/customer-deployments/download/helm/{mapping.id}` endpoint'i çağrılmalı
- Backend, on-the-fly `.tgz` üretip indirme başlatmalı
- `VersionPackage` kaydının DB'de olup olmaması düğmeyi etkilememeli

## Gerçekleşen Davranış

`helmPackage` (VersionPackage kaydı) DB'de yoksa `disabled` kalıyor.
`helmPackage` varsa bile `artifactUrl` null ise `handleDownload(null)` çağrılıyor ve
`if (url) window.open(...)` koruması nedeniyle hiçbir şey olmuyor.

## Teknik Kök Neden

Backlog'da S-01 implementasyonu yazılırken download düğmeleri yeni endpoint'lere
yönlendirilmedi — `VersionPackage.artifactUrl` (eski sistem) referans alındı.
Backend O-02/O-04 endpoint'leri doğru çalışıyor; frontend'de yanlış URL kullanılıyor.

## Önerilen Düzeltme

```tsx
// HELM düğmesi — VersionPackage bağımsız, doğrudan endpoint
<Button
  disabled={false}
  onClick={() => window.open(`/api/customer-deployments/download/helm/${mapping.id}`, '_blank')}
>
  HelmChart İndir
</Button>

// BINARY düğmesi
<Button
  disabled={false}
  onClick={() => window.open(`/api/customer-deployments/download/binary/${mapping.id}`, '_blank')}
>
  Paketi İndir
</Button>
```

> **Not:** Backend açıkken doğrulanmalı — endpoint TS 0 hata ✅
