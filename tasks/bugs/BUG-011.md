# BUG-011 — ReleaseCalendarPage: PRODUCTION versiyonları arşivlenemiyor

## Problem

`PRODUCTION` aşamasındaki versiyonlar için "Arşivle" aksiyonu yoktur.  
UX tasarımında (`designs/screens/release-calendar.md`) PRODUCTION satırında `[Arşivle] [···]` butonu gösterilmektedir.

Mevcut durumda:
- `PHASE_META['PRODUCTION'].next = null` → liste satırında advance butonu gösterilmiyor ✓
- Ancak "Arşivle" inline aksiyonu da mevcut değil ✗
- `ARCHIVED` fazı `PHASE_ORDER` array'inde yer almıyor → `/phase` route'u PRODUCTION'dan ARCHIVED'a geçişi desteklemiyor ✗

## Beklenen Davranış

1. PRODUCTION aşamasındaki versiyonlarda ileri ok yerine `[Arşivle]` butonu görünür
2. Tıklandığında versiyon ARCHIVED fazına geçer
3. Arşivlenen versiyonlar varsayılan listede gizlenir (zaten uygulanmış ✓)

## Teknik Analiz

**Backend:**
```typescript
// productVersions.routes.ts
const PHASE_ORDER = ['PLANNED', 'DEVELOPMENT', 'RC', 'STAGING', 'PRODUCTION'];
// ARCHIVED eklenemez çünkü /phase route'u PHASE_ORDER'ın son elemanını zaten hata fırlatıyor
// → Ayrı bir PATCH /api/product-versions/:id/archive endpoint'i daha temiz
```

**Frontend:**
```tsx
// ListView'da PRODUCTION satırı için:
// nextLabel = null → hiçbir şey gösterilmiyor
// Arşivle butonu eklenmeli
```

## Önerilen Çözüm

**Backend:** `PATCH /api/product-versions/:id/archive` endpoint'i ekle:
```typescript
router.patch('/:id/archive', requireRole('ADMIN', 'RELEASE_MANAGER'), async (req, res, next) => {
  // phase'i ARCHIVED yap
  // isActive = false (eğer varsa)
});
```

**Frontend:** `ListView`'da PRODUCTION için ayrı buton:
```tsx
{v.phase === 'PRODUCTION' ? (
  <Button size="small" variant="outlined" color="default"
    onClick={(e) => { e.stopPropagation(); archiveMut.mutate(v.id); }}>
    Arşivle
  </Button>
) : nextLabel ? (
  <Button ...>→ {nextLabel}</Button>
) : null}
```

## Reproduksiyon

1. `/release-calendar` açılır
2. PRODUCTION aşamasında versiyon bulunur
3. Satırda ne "→ ARC" ne de "Arşivle" butonu görünür
4. Versiyonu arşivlemenin UI yolu yoktur

## Notlar

- P2 önceliği: PRODUCTION versiyonlar listeleniyor, bu bir block değil
- Arşivlenen versiyonlar "Arşivlenenleri Göster" toggle ile görülebilir checkbox zaten var
