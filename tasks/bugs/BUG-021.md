# BUG-021: ReportIssuePage — Dosya yükleme adımı başarısız olunca kullanıcı başarı mesajı görüyor

## Sorun

`submitMutation`'da sorun oluşturma ve dosya yükleme **sıralı** olarak yapılıyor:
1. `POST /transition-issues` → sorun kaydı oluşturulur
2. Her dosya için `POST /transition-issues/:id/attachments/upload` → yükleme yapılır

Eğer adım 2'de herhangi bir dosya yüklemesi başarısız olursa (ağ hatası, boyut limiti aşımı,
server hatası vb.) `await` zinciri exception fırlatır — ancak `onError` callback'i
**sorun oluşturma da dahil her türlü hata** için aynı genel mesajı gösteriyor:
`'Sorun gönderilemedi, lütfen tekrar deneyin.'`

Asıl problem şu: sorun başarıyla oluşturuldu ama dosya yüklenemedi. Kullanıcı "sorun
gönderilemedi" mesajını görünce **tekrar gönderir** → ikinci gönderimde yeni (duplicate) bir sorun
kaydı açılır.

## Etkilenen Kod

```tsx
// packages/frontend/src/pages/ReportIssuePage.tsx ~line 460
const submitMutation = useMutation({
  mutationFn: async (data: typeof form) => {
    // Adım 1: Sorun oluştur
    const res = await apiClient.post('/transition-issues', { ... });
    const issueId: string = res.data.data?.id;

    // Adım 2: Dosya yükle — herhangi biri başarısız olursa exception fırlatır
    if (issueId && attachedFiles.length > 0) {
      for (const file of attachedFiles) {
        const fd = new FormData();
        fd.append('file', file);
        await apiClient.post(`/transition-issues/${issueId}/attachments/upload`, fd, {
          headers: { 'Content-Type': 'multipart/form-data' },
        });
        // ❌ hata olursa throw → onError çalışır ama sorun zaten DB'de var
      }
    }
    return res;
  },
  onSuccess: () => { ... setTab(1); },
  onError: () => setSnackbar('Sorun gönderilemedi, lütfen tekrar deneyin.'),  // ❌ yanıltıcı mesaj
});
```

## Adımlar (Reproduce)

1. Sorun Bildir formunu doldur
2. 20 MB'tan büyük bir dosya ekle (backend limit: `20 * 1024 * 1024`)
3. "Gönder"e tıkla
4. Sorun DB'ye yazılır ama dosya yüklemesi 413 Payload Too Large alır
5. Kullanıcı "Sorun gönderilemedi, lütfen tekrar deneyin." görür
6. Tekrar gönderince duplicate sorun açılır

## Beklenen Davranış

- Sorun başarıyla oluşturuldu, dosya yüklemesi kısmi başarısız olduğunda:
  `"Sorun oluşturuldu fakat 1 dosya yüklenemedi."` gibi ayrıştırılmış mesaj
- Sayfa tab 1'e geçmeli (sorun oluşturuldu)
- Yalnızca sorun oluşturma adımı başarısız olursa "tekrar deneyin" mesajı gösterilmeli

## Gerçekleşen Davranış

Dosya yüklemesi başarısız olduğunda sorun yine de oluşturulmuş durumdadır ancak
kullanıcıya başarısız olduğu (tamamen) söylenir.

## Teknik Kök Neden

`mutationFn` tek bir try-zinciri olarak yazılmış. İki ayrı aşama (issue create + file upload)
hata yönetimi ayrıştırılmamış. Dosya boyutu limiti backend `multer` konfigürasyonunda
`fileSize: 20 * 1024 * 1024 (20 MB)` olarak tanımlı.

## Önerilen Düzeltme

```tsx
mutationFn: async (data: typeof form) => {
  // Adım 1: Sorun oluştur (kritik — başarısız olunca throw et)
  const res = await apiClient.post('/transition-issues', { ... });
  const issueId: string = res.data.data?.id;

  // Adım 2: Dosya yükle (kritik değil — hızlı başarısız olursa uyar, devam et)
  const uploadErrors: string[] = [];
  if (issueId && attachedFiles.length > 0) {
    for (const file of attachedFiles) {
      try {
        const fd = new FormData();
        fd.append('file', file);
        await apiClient.post(`/transition-issues/${issueId}/attachments/upload`, fd);
      } catch {
        uploadErrors.push(file.name);
      }
    }
  }

  return { res, uploadErrors };
},
onSuccess: ({ uploadErrors }) => {
  qc.invalidateQueries({ queryKey: ['transition-issues'] });
  if (uploadErrors.length > 0) {
    setSnackbar(`Sorun oluşturuldu, ${uploadErrors.length} dosya yüklenemedi: ${uploadErrors.join(', ')}`);
  } else {
    setSnackbar('Sorun başarıyla bildirildi.');
  }
  setForm(emptyForm);
  setAttachedFiles([]);
  setTab(1);
},
onError: () => setSnackbar('Sorun gönderilemedi, lütfen tekrar deneyin.'),
```
