# BUG-012 — computePhase: PRODUCTION phase'deki versiyonlar PROD gösterilmiyor

**Tür:** FRONTEND
**Öncelik:** P1
**Ekran:** Release Calendar (`ReleaseCalendarPage.tsx`)
**Tarih:** 2026-02-23

---

## Sorun

`computePhase()` fonksiyonu yalnızca `v.releaseDate` alanına bakıyor.  
Seed/legacy verilerden gelen bazı versiyonlar `phase: 'PRODUCTION'` ama `releaseDate: null`.  
Örnek: versiyon `1.3.1` — phase=PRODUCTION, releaseDate=null.

Sonuç: Bu versiyonlar `computePhase` tarafından PROD değil DEV/WAITING olarak hesaplanır
ve listede yanlış badge + "Yayına Al" butonu gösterir.

---

## Kök Neden

```typescript
// ❌ Yalnızca releaseDate'e bakıyor:
function computePhase(v: Version): string {
  if (v.releaseDate) return 'PROD';
  ...
}
```

API yanıtı:
```json
{ "version": "1.3.1", "phase": "PRODUCTION", "releaseDate": null }
```

---

## Fix

`v.phase === 'PRODUCTION'` kontrolü de eklendi:

```typescript
function computePhase(v: Version): string {
  if (v.releaseDate || v.phase === 'PRODUCTION') return 'PROD';
  ...
}
```
