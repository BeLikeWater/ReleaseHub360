# BUG-015: Release Health Check — `last-prep-releases` 404, Son Release kolonu yüklenmiyor

## Sorun

Release Health Check sayfasında Appwys ürünü seçilip herhangi bir versiyon seçildiğinde,
"Servis Versiyonları" kutusunda **Son Release** sütunu verisi yüklenmiyor.
Arka planda `/api/tfs/last-prep-releases` endpoint'i **HTTP 404** döndürmekte.

## Adımlar (Reproduce)

1. Release Health Check sayfasını aç
2. Ürün olarak **Appwys**'ı seç
3. Bir versiyon seç
4. "Servis Versiyonları" kutusundaki Son Release sütununu incele → "—" görünür
5. Browser DevTools → Network → `last-prep-releases` isteği → 404

## Beklenen Davranış

`/api/tfs/last-prep-releases?productId=...` endpoint'i 200 döndürmeli ve
servislerin Azure VSRM'den son prep release adını getirmeli.
Eğer Azure creds yoksa/ulaşılamıyorsa `{ data: [], authError?: "..." }` dönmeli.

## Gerçekleşen Davranış

HTTP 404 — endpoint hiç bulunamıyor.

## Teknik Kök Neden

`packages/backend` için çalışan eski Node.js prosesi (PID 79914) port 3001'i tutmaktaydı.
Bu proses, `/api/tfs/last-prep-releases` endpoint'i backend koduna eklenmeden **önce** başlatılmıştı.
Yeni başlatma girişimleri `EADDRINUSE: :::3001` hatasıyla başarısız oldu.
`kill -9 79914` komutu çalıştırmakta "operation not permitted" verdi (macOS sandbox / launchd orphan).

**Workaround + Fix:**
- Yeni backend `PORT=3002` ile `packages/backend` dizininden başlatıldı
- `packages/frontend/vite.config.ts` proxy target `http://localhost:3002` olarak güncellendi

```diff
# vite.config.ts
- target: 'http://localhost:3001',
+ target: 'http://localhost:3002',
```

**Kalıcı çözüm:** Makine yeniden başlatıldığında orphan proses ölecek ve her iki backend de
sadece `packages/backend/` dizininden başlatılmalı. Port 3001 tekrar kullanılabilir hale gelecek.
Vite proxy 3001'e geri döndürülmeli.
