# BUG-016: Release Health Check — `moduleGroups.flatMap()` null erişimi crash riski

## Sorun

`ReleaseHealthCheckPage.tsx` içindeki `BomSection` ve `HealthScorePanel` bileşenlerinde
`product.moduleGroups.flatMap(...)` ve `product.services.filter(...)` çağrıları
null-check olmadan yapılmaktaydı.

Backend Prisma `include` ile her zaman `[]` dönse de, tip güvensizliği ve olası backend
değişikliklerinde bu satırlar `TypeError: Cannot read properties of null (reading 'flatMap')`
hatasına yol açar. Bu hata React error boundary'yi tetikler ve sayfanın tamamen boş görünmesine
neden olur.

## Etkilenen Satırlar (Öncesi)

```typescript
// BomSection — allSvcCount (line ~357)
const fromMods = product.moduleGroups.flatMap(g => g.modules.flatMap(m => m.services));
return fromMods.length + product.services.filter(s => !modSvcIds.has(s.id)).length;

// BomSection — JSX render (line ~391)
{product.moduleGroups.map(group => ( ... ))}

// BomSection — ungrouped (line ~412)
const grouped = new Set(product.moduleGroups.flatMap(g => g.modules.flatMap(m => m.services.map(s => s.id))));
const ungrouped = product.services.filter(s => !grouped.has(s.id));

// HealthScorePanel — serviceCount (line ~1089)
const fromMods = productDetail.moduleGroups.flatMap(g => g.modules.flatMap(m => m.services));
return fromMods.length + productDetail.services.filter(s => !modSvcIds.has(s.id)).length;
```

## Adımlar (Reproduce)

1. Backend'de `moduleGroups` null döndürecek bir ürün oluştur
2. Release Health Check → ürünü seç
3. "Servis Versiyonları" kutusu açık — sayfa crash (beyaz ekran veya error boundary)

## Beklenen Davranış

`moduleGroups` null/undefined olsa da sayfa çökmemeli; "Servis yok" veya boş liste göstermeli.

## Gerçekleşen Davranış

`TypeError: Cannot read properties of null (reading 'flatMap')` — sayfanın çökmesi.

## Teknik Kök Neden

`ProductDetail` tipinde `moduleGroups: ModuleGroupRow[]` zorunlu olarak tanımlı,
ancak runtime değeri `null` gelebilir. TypeScript tipi ile runtime gerçeği arasındaki
uyumsuzluk.

## Fix (Uygulandı)

Tüm 4 kullanım noktasına `?? []` null guard eklendi:

```typescript
// Düzeltilmiş - tüm örnekler
const fromMods = (product.moduleGroups ?? []).flatMap(...);
return fromMods.length + (product.services ?? []).filter(...).length;
{(product.moduleGroups ?? []).map(group => ( ... ))}
const grouped = new Set((product.moduleGroups ?? []).flatMap(...));
const ungrouped = (product.services ?? []).filter(s => !grouped.has(s.id));
```

TypeScript: `tsc --noEmit` → 0 hata confirmed.
