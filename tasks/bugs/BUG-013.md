# BUG-013 — ARCHIVED versiyonlar filtrelenmeden aktif liste'de görünüyor

**Tür:** FRONTEND
**Öncelik:** P2
**Ekran:** Release Calendar (`ReleaseCalendarPage.tsx`)
**Tarih:** 2026-02-23

---

## Sorun

`filtered` array hesaplanırken `v.phase === 'ARCHIVED'` kontrolü yok.  
`computePhase()` da ARCHIVED phase'i tanımıyor — tarihlere göre DEV/TEST/PREP/WAITING döndürür.  
Sonuç: Arşivlenmiş bir versiyon takvimde aktif versiyon gibi görünür.

---

## Kök Neden

```typescript
// ❌ ARCHIVED filtresi yok:
const filtered = useMemo(() => {
  return versions.filter((v) => {
    if (!productFilter || v.product.id !== productFilter) return false;
    const phase = computePhase(v);
    if (hideProd && phase === 'PROD') return false;
    return true;
  });
}, ...);

// ❌ computePhase ARCHIVED'ı tanımıyor:
function computePhase(v: Version): string {
  if (v.releaseDate) return 'PROD';
  // ARCHIVED check yok
  ...
}
```

---

## Fix

1. `computePhase` içine `v.phase === 'ARCHIVED'` erken çıkış eklendi
2. `filtered` içinde ARCHIVED default olarak gizleniyor, `hideProd` toggle'ı ile gösterilebilir hale geldi
