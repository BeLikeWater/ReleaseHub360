# BUG-009: CodeSync endpoint'leri MCP server kapalıyken 500 döndürüyor (fetch failed)

## Sorun

Code Sync sayfasındaki tüm işlevler (geçmiş görüntüleme, PR listesi, branch karşılaştırma, sync başlatma)
çalışmıyor. Backend `/api/code-sync/*` endpoint'leri 500 HTTP hatası döndürüyor.

## Adımlar (Reproduce)

1. MCP Python API server kapalı/erişilemez durumda olsun (varsayılan durum: `localhost:8083` çalışmıyor)
2. Login yap
3. Sol menüden "Code Sync Yönetimi" ekranına git
4. Sayfa "MCP Server bağlantısı kurulamadı" uyarısı gösterir
5. Curl ile test et:
   ```bash
   curl -H "Authorization: Bearer $TOKEN" http://localhost:3001/api/code-sync/merged-pr-history
   # → {"error":"Sunucu hatası","details":"fetch failed"}  (HTTP 500)
   ```

## Beklenen Davranış

MCP server erişilemez olduğunda backend **503 Service Unavailable** döndürmeli ve açıklayıcı mesaj ile birlikte frontend'in hangi servisi yapılandırması gerektiğini belirtmeli.

## Gerçekleşen Davranış

HTTP 500 dönüyor. `fetch failed` raw error mesajı client'a sızıyor.

## Teknik Kök Neden

`codeSync.routes.ts` içindeki `mcpRequest` fonksiyonu:
```typescript
async function mcpRequest(path: string, method = 'GET', body?: unknown) {
  const res = await fetch(`${MCP_BASE()}${path}`, { ... });  // ← fetch atarsa yakalanmıyor
  if (!res.ok) throw new AppError(res.status, `MCP Server hatası: ${res.statusText}`);
  return res.json();
}
```

`fetch()` ağ hatası fırlatırsa (connection refused / timeout), `AppError` değil ham JS `Error` fırlatılır.
Bu hata `next(err)` ile genel error handler'a düşer ve `500 Sunucu hatası` olarak döner.

`AppError` sadece HTTP response hatalarını yakalar, ağ bağlantı hatalarını yakalamıyor.

## Önerilen Fix

`packages/backend/src/routes/codeSync.routes.ts`:

```typescript
async function mcpRequest(path: string, method = 'GET', body?: unknown) {
  let res: Response;
  try {
    res = await fetch(`${MCP_BASE()}${path}`, {
      method,
      headers: { 'Content-Type': 'application/json' },
      ...(body ? { body: JSON.stringify(body) } : {}),
    });
  } catch {
    throw new AppError(503, 'MCP Server erişilemez. Ayarlar > Entegrasyonlar bölümünden MCP Server URL\'ini kontrol edin.');
  }
  if (!res.ok) throw new AppError(res.status, `MCP Server hatası: ${res.statusText}`);
  return res.json();
}
```

## Etki

- Code Sync sayfası tamamen işlevsiz (sync geçmişi, PR listesi, yeni sync başlatma)
- Frontend bunu yakalar ve kullanıcıya uyarı gösterir (BUG P2 değil P1'e yükseltilecekse de frontend graceful handle ediyor)
- 500 yerine 503 dönmek API'nın doğru HTTP semantiğini sağlar

---
> **Backend Developer**: `fetch` çağrısını `try/catch` içine al, ağ hatasını `AppError(503, ...)` olarak yeniden fırlat. Aynı pattern `n8n`, `tfs` proxy route'larında da uygulanmalı.
