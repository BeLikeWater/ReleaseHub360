# BUG-018: CustomerDashboardPage — PlanTransitionModal hata durumunda sessiz başarısız oluyor

## Sorun

`PlanTransitionModal` içindeki `createMutation`, `onError` callback'i olmadan tanımlanmış.

Geçiş oluşturma isteği başarısız olduğunda (örneğin aynı `[customerId, toVersionId, environment]`
üçlüsü için unique constraint ihlali, ya da geçersiz `toVersionId` UUID) modal ya:
- "Kaydediliyor..." durumunda takılı kalır
- Ya da hata mesajsız sessizce kapanır

Kullanıcı başarısız işlemi anlamıyor ve tekrar deniyor.

## Etkilenen Kod

```tsx
// packages/frontend/src/pages/CustomerDashboardPage.tsx ~line 139
const createMutation = useMutation({
  mutationFn: async () => {
    const promises = envList
      .filter(env => dates[env])
      .map(env =>
        api.post('/customer-version-transitions', { ... }),
      );
    await Promise.all(promises);
  },
  onSuccess: () => { ... },
  // ❌ onError yok!
});
```

## Adımlar (Reproduce)

1. `PlanTransitionModal`'ı aç
2. Herhangi bir ortam için tarih seç
3. Aynı `[customerId, toVersionId, environment]` kombinasyonu zaten kayıt varsa "Geçişleri Oluştur"a tıkla
4. Backend `409 Conflict` ya da `400 Bad Request` döner
5. Modal'da hiçbir hata mesajı görünmez

## Beklenen Davranış

`onError` callback'inde `setToast('Geçiş oluşturulamadı: ...')` ile hata mesajı gösterilmeli.
`createMutation.isPending` `false`'a dönmeli (düğme tekrar aktif olmalı).

## Gerçekleşen Davranış

- İstek hata alır
- `createMutation.isPending` false'a düşer ama kullanıcı geri bildirim almaz
- Düğme yeniden tıklanabilir hale gelir ancak ne olduğu belirsiz

## Teknik Kök Neden

S-03 implementasyonunda `useMutation` hook'una `onError` handler eklenmedi.
`Promise.all` herhangi bir promise reddedildiğinde tüm sonraki işlemleri de kesiyor —
bu da kısmi oluşturma durumuna (bazı ortamlar kaydedildi, bazıları kaydedilmedi) yol açıyor.

## Önerilen Düzeltme

```tsx
const createMutation = useMutation({
  mutationFn: async () => { ... },
  onSuccess: () => { ... },
  onError: (err: unknown) => {
    const msg = (err as { response?: { data?: { error?: string } } })
      ?.response?.data?.error ?? 'Geçiş oluşturulamadı.';
    // setToast prop olarak geçirilmeli veya yerel state açılmalı
    alert(msg); // geçici — toast prop ile değiştirilmeli
  },
});
```

> **Not:** `PlanTransitionModal`'a `onError` prop eklenmeli ya da `setToast` parent'tan geçirilmeli.
