# BUG-010: ReleaseNotesPage — "Yayınla" butonu 404 hatası veriyor (PATCH endpoint yok)

## Sorun

Release Notları sayfasında bir versiyon seçilip notlar yazıldıktan sonra "Yayınla" butonuna tıklandığında
`404 Not Found` hatası alınıyor. Notlar hiçbir zaman yayına alınamıyor.

## Adımlar (Reproduce)

1. Login yap (ADMIN veya RELEASE_MANAGER rolü)
2. Sol menüden "Release Notları" ekranına git
3. Bir ürün ve versiyon seç
4. Bir veya daha fazla release note ekle
5. "Yayınla" butonuna tıkla

## Beklenen Davranış

`notesPublished` alanı `true` olarak güncellenmeli; buton "✅ Yayında" olarak değişmeli.

## Gerçekleşen Davranış

404 Not Found hatası alınıyor. `notesPublished` alanı `false` kalıyor.

## Teknik Kök Neden

`ReleaseNotesPage.tsx` içindeki `publishMutation`:
```typescript
const publishMutation = useMutation({
  mutationFn: () => api.patch(`/product-versions/${versionId}`, { notesPublished: true }),
  ...
});
```

Backend `productVersions.routes.ts` içindeki mevcut `PATCH` endpoint'leri:
```typescript
router.patch('/:id/phase', ...)    // ← PATCH /product-versions/:id/phase  ✓ var
// PATCH /product-versions/:id           ← YOK ❌
```

`PATCH /product-versions/:id` → 404 döner çünkü bu route tanımlı değil.
`PUT /product-versions/:id` var ama genel versiyon update'i için, `notesPublished` alanını kabul edip etmediği belirsiz.

## Önerilen Fix (2 Seçenek)

**Seçenek A — Backend: Yeni endpoint ekle (Önerilen)**

`packages/backend/src/routes/productVersions.routes.ts`:
```typescript
// PATCH /api/product-versions/:id — genel alan güncellemesi (notesPublished v.b.)
router.patch('/:id', requireRole('ADMIN', 'RELEASE_MANAGER'), async (req, res, next) => {
  try {
    const { notesPublished, description } = req.body;
    const updated = await prisma.productVersion.update({
      where: { id: req.params.id },
      data: { ...(notesPublished !== undefined && { notesPublished }), ...(description && { description }) },
    });
    res.json({ data: updated });
  } catch (err) {
    next(err);
  }
});
```

**Seçenek B — Frontend: Mevcut PUT endpoint'ini kullan**

Bu seçenek daha riskli çünkü `PUT` tüm alanları zorunlu kılabilir.

## Etki

- Release notları **asla yayına alınamıyor**
- Müşteri portalı veya dış sistemlere notlar yayınlanamamış olacak
- `notesPublished` alanı `ReleasesPage`'de de gösteriliyor (Chip: "Evet/Hayır"), yanlış veri gösteriyor

---
> **Backend Developer**: Seçenek A tercih edilmeli. `PATCH /:id` route'unu `PATCH /:id/phase` route'unun üstüne ekle. `notesPublished` ve opsiyonel `description` alanlarını güncelleyebilmeli.
